Getting started with a project

git init
Ask Git to initialize the folder as a Git project
git status
Shows which files are staged (the files Git has indexed and up-to-date), and which are not staged (the files Git has not indexed up-to-date).
git add path/to/file
To add a file to the staging area (Git indexes the file, i.e. makes a note of the file contents. We need to keep running this command every time we make a change to the file.
git add .
Stage the files in the current folder and all sub-folders
git commit -m "commit message as per the work you have done"
Creates a "commit", a "version" of your project codebase
git log
Gives us the commit history
git remote add origin url_of_origin_remote
To set up the Git repo to push the code to a remote repo. Here origin is NOT a keyword. It is a common name given for the default remote.
git remote -v
To check the remote repositories tied up to your local repo
